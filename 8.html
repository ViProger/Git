Родственные связи тегов

Разберемся с родственными связями тегов. Это значит что каждый тег может быть либо родителем либо потомком либо одновременно и тем и другим относительно других.

Рассмотрим следущий код для примера

<ul> <li>пункт <i>курсив</i></li> <li>пункт <i>курсив</i></li> <li>пункт <i>курсив</i></li> <li>пункт <i>курсив</i></li> </ul>

Обратив внимание на расположение тега i мы можем сделать следующие выводы. li и ul для него являются родителями, в тоже время для тега li родителем является только ul в вот тег i для него потомок.

В этом ничего особо сложного нет, полагаю каждый уже понял как определить родственность тегов. По этому перейдем к следующему пункту

Наследование CSS свойств

Наследование свойств CSS происходит от родителя к потомку, тоесть если мы с ледующем коде покрасим тег р то и тег i унаследует это свойство, пока мы не изменить его либо не изменим конкретно для него.

<p> какой-то <i>текст</i> в абзаце </p> p { color: red; }

Группировка селекторов CSS

Селекторы в CSS можно соединять в группу, это нужно если например нам нужно сразу нескольким тегам задать одинаковые свойства. Например:

p, i, h1 { color: green; }

Это тоже самоче что и:

p { color:green; } i { color:green; } h1 { color:green; }

По мимо этого такой способ можно применять не обязательно к обсолютно одинаковым тегам. То есть, если 3 тега имеют одинаковые свойства цвета, а один из них имеет еще и другой размер или начертание, то это можно следать просто сгрупировав одинаковые свойства и дополнив затем нужный тег отдельным свойством которое будет относиться только к нему. Например:

h2, h3, P { color: red; } P { text-align: justify; font-size: 16px; }

Селектор потомков на CSS

Например у нас есть два типа списков

<ul> <li>пункт</li> <li>пункт</li> <li>пункт</li> <li>пункт</li> <li>пункт</li> </ul> <ol> <li>пункт</li> <li>пункт</li> <li>пункт</li> <li>пункт</li> <li>пункт</li> </ol>

И мы хотим покрасить все теги li в красный цвет, то мы просто напишем следующее в css

li {color: red;}

А что если нам нужно сделать два списка с разными цветами пунктов. Вот здесь то мы и будем указывать потомственную связь тегов, которая даст коду понять какие конкретно пункты li красить в один цвет а какие в другой. Выглядит это следующим образом

ul li {color:red;} ol li {colore: green;}

если сказать иначе. В поставленной задаче нам поможет селектор потомков. Он позволяет выбирать теги по их родителю. Для этого нужно указать селектор родителя, а через пробел - селектор потомка. В нашем случае селектор ul li выберет все теги li из списка ul, а селектор ol li - выберет все теги li из списка ol.

Помимо всего, селектор потомков может состоять из бесконечного количества, все зависит от степени вложения тегов в теги или родственности тегов.

Дочерний селектор в CSS >

Для простоты понимая просто приведем пару примеров того в чем разница между потомками и конкретно дочерним наследованием.

Допустим есть код:

<p> текст <b><i>жирный курсив</i></b> </p> <p> текст <i>просто курсив</i> </p>

В данном случае если нам нужно изменить все теги i мы делаем следующее

p i { color: red;}

А что если нам нужно изменить только те i которые являются прямыми потомками тега р (тоесть дочками родителей напрямую). В таком случае используем дочерний селектор.

p > i { color: red;}

В этом случае те i который находится внутри тега b не изменится, так как он не дочерний к тегу p, а скажем внучатый, он является внучкой а не дочкой.

Классы тегов в CSS >

В предыдущих уроках мы с вами выбирали элементы страницы по имени тега, задавая стили, к примеру, одновременно всем абзацам. На странице, однако, могут быть абзацы разных типов, к которым нужно применять разные стили. Для решения такой проблемы можно разные абзацы отнести к разным CSS классам. Чтобы отнести тег какому-нибудь классу, нужно этому тегу написать атрибут class, а в нем - придуманное вами название класса, состоящее из букв, цифр, символов подчеркивания и дефиса.

Атрибуты тегов указываются внутри открывающегося тега. Тоесть

<p атрибут = "значение атрибута" > </p> <p class = "название_класса" > </p>

Чтобы обратится к тегу с конкретным классом в CSS необходимо перед именем класса поставить точку. Или же записать полную структуру тег.имя_класса

Например:

.class {color:green;} или полная записать p.class {color: red}

Тег span для кусочков текста

Вы уже знаете, что для того, чтобы сделать некоторый кусочек текста, к примеру, жирным - следует использовать тег b. Однако, что делать, если я хочу покрасить кусочек текста в красный цвет? Для такого случая тега, подобного тегу b, не существует. Зато существует специальный тег span - сам по себе он абсолютно ничего не делает, но ему можно указать класс и применить CSS стили, которые позволят нам добавить нужный эффект к кусочку текста.

Например:

<p> какой-то <span class="red">текст</span> в абзаце </p> .red { color: red; }

Выбор элемента по уникальному идентификатору ID на CSS

Помимо классов существует определение элемента по его уникальному идентификатору ID, разница классов от ID как раз в названии УНИКАЛЬНЫЙ идентификатор, тоесть элемент с таким ID может быть только один, в то время как элементов с одинаковым классом может быть множество.

Так же можно к тегу с ID добавлять классы как и прежде. Например какой нибудь уникальный элемент оформления сделать через ID а все остальное добавлять с помощью классов.

Чтобы обратится к селектору по ID нужно вместо точки использовать октоторп (хештег, шарп) #

Так же следует учесть что с ID все так же как и с классами, его основное отличие он присваивается только одному элементу.

Чтобы выбрать элемент у которого два класса, но при этому не затронуть другие элементы, которые используют эти классы по отдельности, нужно написать два этих класса слитно без пробела

<p class="eee"> абзац </p> <p class="zzz"> абзац </p> <p class="eee zzz"> абзац </p> .eee.zzz { color: red; }

Группировка сложных селекторов

Пусть у нас есть два следующих селектора:

.block h2 { color: red; } .block h3 { color: red; } Так как эти селекторы делают одно и тоже их можно сгруппировать через запятую: .block h2, .block h3 { color: red; }

Приоритет селекторов CSS

есть следующий код:

p { color: red; } p { color: green; }

Коротко говоря в такой ситуации, то что идет ниже (или скажем последним), то и применяется. Это относится только к конфликту свойств. То есть к случаю, когда и в одном селекторе, и во втором есть одинаковые свойства. Если свойства разные, то просто применятся свойства и одного, и второго селектора.

p { font-size: 20px; } p { color: green; }

Применится и первое и второе свойство.

Правило Специфичности

Бывает так что одному элементу соответствует два селектора. p и например div p. В такой ситуации побеждает более точный селектор в данном случае div p.

Основные правила приоретизации:

  1. Класс всегда побеждает селктор тега
  2. ID Всегда побеждает класс
  3. При равных условиях побеждает тот селектор который состоит из большего количества частей

Соседний селектор

Соседний селектор + позволяет выбрать элемент по его соседу сверху. Тоесть он выбирает все теги которые иду сразу после тега который указан перед плюсом. Пример:

<h2>text</h2> <p> + </p> //Будет выбран этот параграф <p> - </p> <p> - </p> <h2>text</h2> <p> ++ </p> //Будет выбран этот параграф <p> - </p> <p> - </p> h2 + p { color: red; }

Обратимся ко всем тегам с классом elem после элемента с классом blok. Пример:

<h2 class="blok">text</h2> <p> + </p> //Будет выбран этот параграф <p class="elem"> - </p> <p> - </p> .blok + .elem { color: red; }

Родственный селектор в CSS

Родственный селектор ~ позволяет выбрать все элементы, находящиеся после заданного элемента внутри одного родителя.

Давайте обратимся ко всем тегам p, находящимся после тегов h2, и покрасим их в красный цвет:

<div> <h2>text</h2> <p> + </p> <p> + </p> <p> + </p> </div> <p> - </p> h2 ~ p { color: red; }

Атрибут со значением

Чтобы задать свойства для атрибу с конкретным значением используется следующий синтаксис.

[Атрибут="Значение"] {Описание правил стиля} или Селектор[Атрибут="Значение"] {Описание правил стиля}

Значение атрибута начинается с определённого текста

[Атрибут^="Значение"] {Описание правил стиля} или Селектор[Атрибут^="Значение"] {Описание правил стиля}

Значение атрибута оканчивается определённым текстом

[Атрибут$="Значение"] {Описание правил стиля} или Селектор[Атрибут$="Значение"] {Описание правил стиля}

Значение атрибута содержит определённый текст

[Атрибут*="Значение"] {Описание правил стиля} или Селектор[Атрибут*="Значение"] {Описание правил стиля}

Обращаемся к элементу по одному из значений атрибута (например есть два класса, обратимся по одному из них)

[Атрибут~="Значение"] {Описание правил стиля} или Селектор[Атрибут~="Значение"] {Описание правил стиля}

Дефис в значении атрибута

В именах идентификаторов и классов разрешено использовать символ дефиса (-), что позволяет создавать значащие значения атрибутов id и class. Для изменения стиля элементов, в значении которых применяется дефис, следует воспользоваться следующим синтаксисом.

[атрибут|="значение"] { Описание правил стиля } или же Селектор[атрибут|="значение"] { Описание правил стиля }
